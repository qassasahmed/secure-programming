OWASP Secure File Handling Guidelines

  • Input Validation and Sanitization: Validate file names, check extensions and MIME types, limit file size, and sanitize user input.
  • Secure File Storage: Store files outside the webroot, use randomized file names, and isolate file storage.
  • Access Control: Enforce authentication, restrict file permissions, and use secure download mechanisms.
  • Secure File Uploads: Scan uploaded files, avoid executing them, and use secure APIs.
  • Prevent Path Traversal: Canonicalize file paths, use chroot jails, and validate directory access.
  • Secure File Processing: Avoid dynamic file inclusion, handle metadata safely, and use trusted libraries.
  • Logging and Monitoring: Log file operations and monitor for anomalies.
  • Client-Side Considerations: Educate users and enforce HTTPS for file transfers.

For more details, visit the OWASP File Upload Cheat Sheet.
